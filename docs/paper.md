# Trident-AE 流式入侵检测与跨年度迁移实验研究

## 摘要

针对网络流量持续演化、未知攻击频繁出现以及跨年份分布漂移带来的检测退化问题，本文基于 Trident 思想实现并评估了一套流式异常检测框架（Trident-AE）。该框架由三部分组成：`tSieve`（一类自编码器学习器集合）、`tScissors`（基于 EVT/POT 的自适应阈值）与 `tMagnifier`（未知样本缓存与 DBSCAN 聚类新类发现）。在 `outputs/runs` 当前批次实验中，我们重点评估 2017→2019 对齐数据上的风险误报率（良性误报）与风险漏报率（风险流量误收为良性）之间的权衡。结果表明：在当前代码与参数空间下，`20260509_151927` 取得最低风险漏报（9.57%）但伴随较高风险误报（11.04%）；`20260509_144554` 取得最低风险误报（0.03%）但风险漏报升至 47.40%；`20260509_164622` 与 `20260509_174929` 给出了更可部署的平衡点（风险误报约 1.5%~~1.8%，风险漏报约 12.1%~~12.5%）。本文进一步结合代码路径解释了该折中关系形成的机理，并给出后续优化方向。

**关键词**：流式入侵检测，开放集识别，增量学习，自编码器，极值理论，未知类发现

## 1. 引言

传统离线入侵检测模型通常假设训练与部署阶段同分布，且类别集合固定。然而真实网络环境中，攻击模式和协议行为会持续变化，表现为以下三类挑战：

1. **新攻击类型不断出现，训练集不可覆盖。**
  例如，模型在 2017 年流量上只学习过 `DOS HULK`、`PORTSCAN` 等模式，当 2019 年出现 `DRDOS_`* 或 `UDP-LAG` 这类新家族时，传统闭集分类器往往只能“硬分到旧类”，导致误判或漏报。
2. **时间漂移导致“已知类别”分布偏移。**
  例如，同样标记为 `BENIGN` 的业务流量，随着业务系统升级、协议栈变化或负载模式变化，其统计特征分布会发生迁移；如果模型阈值固定，容易把新时期正常流量当作异常，或把新型异常当作正常。
3. **误报与漏报存在显著业务权衡。**
  例如，在 SOC 场景中，如果阈值过宽，告警量会下降但真实攻击更容易漏检；如果阈值过严，虽然漏报降低，但告警风暴会让分析人员疲于处置，最终同样影响整体防护效果。

为应对上述挑战，本文聚焦一个可在线更新、可发现新类的流式框架。我们以工程可复现实现为基础，围绕当前 `runs` 实验回答以下问题：

- 当前实现是否可以在跨年度迁移场景下稳定运行？
- 风险误报率与风险漏报率的可达边界在哪里？
- 哪些代码机制导致了当前性能上限与瓶颈？

## 2. 相关工作与理论基础

### 2.1 基于 U-Net 思想的自编码器（AE）

自编码器是异常检测中常见的一类重构模型：通过学习“正常模式”的低维表示与重构映射，利用重构误差区分已知分布与异常分布。  
Trident 体系中的 `tSieve` 采用多学习器一类建模，每个学习器服务于一个类别/簇。在网络结构层面，`trident_stream/tsieve.py` 的 `AutoEncoder` 使用了编码器-解码器对称结构，并在解码阶段引入 skip addition（类似 U-Net 的跨层信息回传思想）。

这种结构的价值在于：

1. **保留细粒度特征**：跨层连接缓解深层压缩造成的信息丢失；
2. **提升训练稳定性**：梯度传播路径更短，增量训练更稳；
3. **增强类别分离能力**：在多学习器并行时，不同学习器可对不同局部模式形成更清晰重构边界。

因此，基于 U-Net 思路的 AE 在 Trident 中不是“图像语义分割”的直接复用，而是其“跨层特征融合”思想在流式重构检测中的迁移应用。

### 2.2 基于 DBSCAN 的未知类聚类发现

DBSCAN 是经典密度聚类算法，能够在未知簇数场景下识别高密度区域并天然过滤噪声点。Trident 的 `tMagnifier` 用其处理未知缓存中的样本，并将达到规模阈值的簇提升为新学习器。

选择 DBSCAN 的原因主要包括：

1. **无需预设簇数**：适合开放环境中的新类发现；
2. **对离群点鲁棒**：可将零散未知样本保留为噪声，不强行归类；
3. **便于在线触发**：与缓存触发机制（`cluster_trigger_size`）易于组合。

同时，DBSCAN 也引入关键超参数敏感性：`eps` 决定邻域半径，`min_samples` 决定核心点密度门槛。本文后续敏感性实验已验证，`eps` 的变化会直接改变“误报-漏报”平衡。

### 2.3 基于 EVT/POT 的阈值评估

阈值估计是重构式检测中的核心环节。固定阈值在流式场景下容易失效，因此 Trident 的 `tScissors` 使用极值理论（EVT）中的 POT（Peak Over Threshold）框架：先选取高分位阈值 `u`，再对超过 `u` 的尾部样本用广义帕累托分布（GPD）拟合，从而估计风险控制意义下的动态阈值。

该思路的理论优势包括：

1. **尾部分布建模能力强**：更贴合异常检测“关注极端值”的目标；
2. **风险可解释**：`evt_risk` 可直接映射为可控的尾部接受概率；
3. **可回退机制**：当样本不足或拟合不稳定时回退到分位数阈值，提升工程鲁棒性。

在本项目实现中，这一机制使阈值不再固定于单次离线训练结果，而是可随学习器状态变化动态刷新，是支撑流式部署的关键基础。

## 3. 方法

### 3.1 系统总体流程

实验入口为 `main.py`（兼容别名 `stream_trident_ae.py`），调用 `TridentStreamingExperiment` 完成端到端流程。其运行逻辑如下：

1. 读取并按时间排序网络流；
2. 使用初始阶段样本建立初始学习器；
3. 按窗口逐样本分类：被已有学习器接受则归类，否则进入未知缓存；
4. 当未知缓存满足触发条件时聚类并创建 `NEW_x` 学习器；
5. 对已有学习器执行增量更新并刷新阈值；
6. 输出指标、日志、学习器画像与时间序列结果。

### 3.2 tSieve：一类自编码器学习器集合

`trident_stream/tsieve.py` 中，`AutoEncoder` 采用对称编码-解码结构，并在解码阶段使用 skip addition。每个学习器以重构误差为判别依据，`classify_sample()` 会优先选择通过阈值的非 BENIGN 学习器，从机制上降低攻击流量误入 BENIGN 的概率。该“非 BENIGN 优先”策略是降低风险漏报的关键设计之一。

### 3.3 tScissors：EVT 阈值估计

`trident_stream/tscissors.py` 使用 POT（Peak Over Threshold）建模尾部分布。对于重构误差样本量足够的学习器，通过广义帕累托分布估计阈值；样本不足或拟合失败时回退到分位数阈值。这使阈值能够随学习器状态动态适配，而非使用固定阈值。

### 3.4 tMagnifier：未知缓存与新类发现

`trident_stream/tmagnifier.py` 中，未知样本先进入缓存，达到 `cluster_trigger_size` 后执行 DBSCAN；满足 `new_class_min_size` 的簇将被提升为新学习器。该机制在开放环境下提供了“未知-新类”闭环，但也带来学习器数量膨胀与类别混叠风险。

## 4. 实验

### 4.1 数据集介绍

基于当前运行配置，主要评估两类场景：

- **同年场景（2017 五天流量）**：`monday.csv`~`friday.csv`；
- **跨年度迁移场景（2017→2019 对齐）**：`aligned_2017_2019_sampled_year_tagged.csv`。

我们重点分析你当前最关注的一组迁移实验（`20260509_164622` 与 `20260509_174929`）及两组对照实验（`20260509_151927` 与 `20260509_144554`）。

### 4.2 指标评估

本项目 `metrics.json` 记录两个核心风险指标：

- **风险误报率（risk_false_positive_rate）**：良性流量被分配到风险学习器的比例；
- **风险漏报率（risk_false_negative_rate）**：风险流量被分配到 BENIGN（或最终未有效拦截）的比例。

两者分别对应“告警噪声”和“漏检风险”，业务上通常不可同时极小。

### 4.3 实验设置（关键参数）

在迁移组中，核心参数为：

- `stream.init_known_mode = benign_only`
- `stream.init_ratio = 0.2`
- `tmagnifier.cluster_trigger_size = 120`
- `tmagnifier.max_unknown_buffer = 30000`
- `tmagnifier.dbscan_eps = 1.2`
- `tmagnifier.dbscan_min_samples = 15`
- `tmagnifier.new_class_min_size = 80`

`20260509_151927` 的关键差异为 `dbscan_eps = 4`（更大邻域），`20260509_144554` 为 `dbscan_eps = 1.0`（更小邻域）。

### 4.5 实验二：学习器演化与未知缓冲信号

从 `run.log` 可见：

- `20260509_164622`：新增学习器 34 个（含初始 BENIGN 共 35），`UnknownFinalize` 仍有 9011；
- `20260509_174929`：新增学习器 32 个（含初始 BENIGN 共 33），`UnknownFinalize` 仍有 8503；
- `20260509_151927`：最终仅 13 个学习器，未知残留 460；
- `20260509_144554`：最终 20 个学习器，但未知残留 9454。

这说明“学习器数量多”并不必然降低未知残留；`dbscan` 聚类形态与接受阈值共同决定未知样本去向。

### 4.6 实验三：迁移场景下的类别可分性观察

以 `20260509_174929/learner_label_distribution.csv` 为例：

- `2017|BENIGN` 学习器纯度约 87.36%，已能维持较高稳定性；
- 多个 `2017` 攻击簇（如 `DOS HULK`, `DOS GOLDENEYE`, `BOTNET-ATTEMPTED`, `DDOS`）形成高纯度 `NEW_x`；
- `2019` 侧部分簇出现混叠（例如 `DRDOS_LDAP/MSSQL/SSDP`、`DRDOS_SNMP/UDP/UDP-LAG`）；
- `UNKNOWN` 桶仍显著（8503），且包含 2017 与 2019 混合样本，表明跨年度域偏移尚未完全吸收。

在 `rows_used=105398`、`init_known_mode=benign_only`、`benign_accept_scale=0.5`、`historical_sample_rate=0`、`dbscan_min_samples=15`、`cluster_trigger_size=120`、`new_class_min_size=80` 基本一致的前提下，仅扫描 `dbscan_eps`，结果如下：

| 20260509_145114_baseline | 1.2        | 0.61%  | 19.05% | 32     |

可以观察到明显趋势：随着 `dbscan_eps` 增大，系统更容易将未知样本并入新/已有风险学习器，**风险漏报率下降**，但同时 **风险误报率显著上升**。  
其中 `eps=2.0`~`2.5 `区间表现出相对可接受折中；`eps=1.0 `过于保守（漏报高），`eps=4.0` 过于激进（误报高）。

### 4.8 实验五：超参数敏感性（历史重训参数）

历史重训相关超参数均配置于 `tsieve`，其含义如下：

- `historical_sample_rate`：每次参数更新时，从历史数据中采样参与训练的比例（0 表示不使用历史样本，仅用当前批数据；0.5 表明采样一半来自历史缓冲）
- `history_samples_per_update`：每次参数刷新时最多采集的历史样本数上限（防止超大规模重训导致计算资源溢出）
- `benign_history_confidence_scale`：历史 BENIGN 样本参与判定时置信度缩放系数，用于调整其对模型阈值的影响权重（1.0 为等权，<1 会降低历史 BENIGN 样本的重要性）

在 `rows_used=80398`、`init_known_mode=benign_only`、`dbscan_eps=0.9` 等其它主参数保持一致的实验簇下，比较不同历史重训配置的效果（主要检索各 run 日志末尾 `Done. METRICS ...` 报告）如下表所示：


| Run ID          | historical_sample_rate | history_samples_per_update | benign_history_confidence_scale | 风险误报率 | 风险漏报率   | 最终学习器数 |
| --------------- | ---------------------- | -------------------------- | ------------------------------- | ----- | ------- | ------ |
| 20260509_105937 | 0.0                    | 0                          | 1.0                             | 0.17% | 35.71%  | 14     |
| 20260509_110713 | 0.0                    | 0                          | 1.0                             | 0.18% | 34.64%  | 17     |
| 20260509_111236 | 0.5                    | 2000                       | 1.0                             | 0.00% | 100.00% | 1      |
| 20260509_111859 | 0.5                    | 2000                       | 1.0                             | 0.00% | 97.84%  | 3      |
| 20260509_113613 | 0.1                    | 300                        | 0.6                             | 0.00% | 96.04%  | 6      |


**结果分析：**

- **无历史重训（前两组）**：即 `historical_sample_rate=0`，表现为风险误报率非常低（约 0.17~0.18%），但风险漏报率较高（约 35%），学习器数中等（14/17 个）。
- **高比例历史重训（后三组）**：如 `historical_sample_rate=0.5`、`history_samples_per_update=2000`，风险误报率虽然降至 0%，但漏报率几乎接近 100%、最终仅剩极少/甚至单一学习器，体现为模型误将绝大多数流量统一归为 BENIGN。`benign_history_confidence_scale` 即使缩小至 0.6，依然无法有效抑制此效应。

上述现象表明**当前实现下，直接提升历史样本采样比例会导致严重退化**：主要是因为历史样本未经过充分筛选，与阈值刷新机制联动不足，导致在增量学习过程中模型快速“同化”，绝大多数历史与新观测流量被合并至 BENIGN 学习器（本质上是过拟合历史 BENIGN 模式，对新风险类别丧失敏感性）。这带来的直接后果是学习器数变少，漏报风险显著上升，检测能力丧失。

**结论及建议：**

- 当前如无针对性样本筛选/阈值动态修正机制，**不建议上线高比例历史重训**（即 `historical_sample_rate ≥ 0.1` 时需慎用，更不推荐直接配合大批量回放如 `history_samples_per_update=2000`）。
- 建议后续配合更精细的历史采样机制和自适应阈值更新，再启用历史重训功能，以充分发挥其对跨域泛化能力的正向作用，并规避上述“过度同化”问题。

### 4.10 实验七：学习器路由优先级消融（攻击优先 vs 并行竞争）

## 5. 讨论、局限与结论

### 5.1 指标权衡的机制解释

基于代码与结果，当前“误报-漏报跷跷板”主要来自三处：

1. `**classify_sample()` 接受机制**
  非 BENIGN 优先可压低漏报，但当新学习器泛化边界较宽时会抬高误报。
2. `**tMagnifier` 聚类粒度（尤其 `dbscan_eps`）**
  `eps` 过大时更易形成大簇并快速承接未知，漏报下降但误报上升；`eps` 过小时未知残留高，误报下降但漏报可能上升。
3. **增量更新策略**
  历史重训并非必然增益；本批敏感性实验显示，当前实现下高比例历史回放会造成严重漏报劣化，提示历史样本筛选与阈值刷新策略仍需改造。

### 5.2 当前最佳实践（基于现有 runs）

若目标是上线可用而非单指标极值，建议以 `20260509_164622` 为新基线，理由：

- 在 16 万级样本上维持较低双风险（1.81% / 12.11%）；
- 学习器扩展充分（35 个），对多攻击族覆盖更完整；
- 与 `20260509_174929` 结果接近，说明配置具备复现稳定性。

### 5.3 威胁与局限

1. 当前评估主要基于项目自定义风险指标，缺少统一 ROC/PR 曲线与成本敏感评价；
2. 跨年度实验中未知桶仍大，说明域对齐与增量记忆机制仍需增强；
3. 学习器数随时间增长较快，部署维护成本（模型数量、更新频率、告警解释）需进一步控制；
4. 部分 2019 攻击族仍存在类别混叠，提示需要更细粒度特征或分层聚类策略。

### 5.4 结论与后续工作

本文基于当前代码与最新实验结果，验证了 Trident-AE 在流式开放场景中的可行性。系统能够通过“未知缓存→聚类→新学习器”机制持续吸收新模式，并在跨年度任务上达到可部署的风险平衡（以 `20260509_164622`/`20260509_174929` 为代表）。与此同时，误报与漏报仍存在结构性权衡，且未知残留和类别混叠提示系统尚未完全收敛。

后续建议：

1. 在 `tsieve` 中恢复历史采样（`historical_sample_rate > 0`），评估跨域稳定性提升；
2. 围绕 `dbscan_eps`、`new_class_min_size` 做小范围网格搜索，目标最小化综合风险 `R = alpha * FPR + beta * FNR`；
3. 对 `UNKNOWN` 增加二次聚类或主动学习复核闭环；
4. 在评估层引入统一曲线指标与运行时资源开销指标，形成“检测效果 + 工程成本”的双目标优化。

### A.1 相关工作算法细节：基于 U-Net 思想的自编码器（AE）

本项目 `tSieve` 采用一类学习器（one-class learner）思路：每个学习器只拟合一个类别/簇的“可重构区域”，通过重构误差判断样本是否属于该学习器。其核心网络在 `trident_stream/tsieve.py` 中实现，结构可抽象为：

- 编码器：`d -> 256 -> 128 -> 64 -> 32`
- 解码器：`32 -> 64 -> 128 -> 256 -> d`
- 跳连方式：解码层与同尺度编码层做逐元素相加（skip addition）

与经典 U-Net 的联系在于“跨层信息直连”思想，而非图像分割任务本身。这里的主要作用是：

1. 在压缩表征过程中保留细粒度统计特征，减少重构时的信息丢失；
2. 缓解深层训练中的梯度衰减，使增量更新更稳定；
3. 对同分布样本形成低损失流形，对异分布样本产生较高重构误差，便于阈值判别。

对单个学习器 L_k，若输入样本为 x，其重构输出为 \hat{x}，则重构误差定义为：

e_k(x)=\frac{1}{d}\sum_{j=1}^{d}(x_j-\hat{x}_j)^2

若 e_k(x)\le \tau_k（\tau_k 为该学习器阈值），则样本被该学习器接受。

### A.2 相关工作算法细节：DBSCAN 未知聚类

`tMagnifier` 将未被任何学习器接受的样本写入未知缓存，达到触发规模后执行 DBSCAN。DBSCAN 的两类核心判定如下：

1. **核心点（core point）**：在半径 `eps` 邻域内，样本数不少于 `min_samples`；
2. **密度可达（density-reachable）**：从核心点出发，经邻域扩展可达的点属于同一簇。

本项目中 DBSCAN 参数与实现对应关系为：

- `dbscan_eps`：邻域半径，控制“簇连接松紧度”；
- `dbscan_min_samples`：核心点最小样本数，控制噪声过滤强度；
- `new_class_min_size`：簇提升为新学习器前的最小规模约束；
- `cluster_trigger_size`：触发一次聚类的未知缓存阈值。

执行流程为：

1. 对未知缓存样本标准化；
2. 运行 DBSCAN，得到簇标签；
3. 过滤噪声簇与过小簇；
4. 将满足规模条件的簇提升为 `NEW_x` 学习器；
5. 从未知缓存移除已消费样本。

这一定义天然支持开放集场景：未知流量不会被强制归为既有类别，而是先缓存、再聚类、再决策。

### A.3 相关工作算法细节：EVT/POT 阈值评估

固定阈值难以适配流式分布漂移，`tScissors` 使用极值理论（EVT）中的 POT（Peak Over Threshold）建模误差尾部。设学习器在参考样本集上的误差为 e_i_{i=1}^{n}，步骤如下：

1. 取高分位基准阈值 u=Q_{\alpha}(e)（由 `evt_quantile` 给定，如 0.95）；
2. 构造超阈值样本 y_i=e_i-u,\ e_i>u，得到尾部样本集合 y_i_{i=1}^{N_u}；
3. 用广义帕累托分布（GPD）拟合 y_i，得到形状参数 \xi 与尺度参数 \beta；
4. 按风险参数 `evt_risk`（记为 q）反推最终阈值 \tau。

在 POT 框架下，阈值估计公式为：

- 当 \xi \neq 0 时：

\tau = u + \frac{\beta}{\xi}\left[\left(\frac{nq}{N_u}\right)^{-\xi}-1\right]

- 当 \xi \to 0 时（指数分布极限）：

\tau = u - \beta\ln\left(\frac{nq}{N_u}\right)


这与 `trident_stream/tscissors.py` 的两条实现分支严格对应：`abs(c) > 1e-6` 时采用前式，否则采用对数形式。  
此外，实现里还加入了多层稳定性保护：

1. 若有效误差样本太少（如 `<100`），直接返回分位数阈值；
2. 若超阈值样本数不足（如 `<30`），回退 `fallback_quantile`；
3. 若 GPD 拟合异常或出现非有限值，回退 `fallback_quantile`；
4. 对 EVT 输出设置下界保护：\tau \ge Q_{0.99}(e)，避免阈值过低导致误报激增。

该设计的工程意义是：阈值会随着学习器当前误差分布动态刷新，并通过 `evt_quantile / evt_risk / fallback_quantile` 三组参数实现风险可调，从而在“过松（漏报高）”与“过严（误报高）”之间维持可控边界。

### A.4 三类算法在 Trident 中的协同关系

三类算法在系统中形成闭环：

1. **AE（tSieve）负责判别前端**：给出样本级重构误差；
2. **EVT（tScissors）负责阈值后端**：把误差映射到可控接受边界；
3. **DBSCAN（tMagnifier）负责未知扩展**：将拒识样本转化为新学习器。

因此，Trident 的能力不来自单一算法，而来自“重构判别 + 尾部阈值 + 未知聚类扩展”的联动机制。该联动也是本文在主文实验部分进行超参数敏感性分析的理论基础。
